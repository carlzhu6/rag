## 📤 文档上传与处理流程

```mermaid
sequenceDiagram
    participant U as 用户浏览器
    participant C as DocumentController
    participant S as DocumentService
    participant T as Apache Tika
    participant E as EmbeddingModel<br/>(Ollama nomic-embed-text)
    participant M as Milvus
    participant DB as MySQL

    U->>C: POST /api/documents/upload<br/>(multipart/form-data)
    C->>C: 验证 JWT Token
    C->>S: uploadDocument(file, userId)

    S->>S: 保存文件到本地磁盘<br/>(uploads/目录)
    S->>DB: 插入文档记录<br/>(status=PARSING)

    S->>T: 解析文档内容<br/>(PDF/Word/TXT/MD)
    T-->>S: 返回纯文本

    S->>S: 文本分块<br/>(500字/块，50字重叠)

    loop 每个文本块
        S->>E: 生成 Embedding 向量
        E->>E: 调用 Ollama API<br/>POST http://localhost:11434/api/embeddings
        E-->>S: 返回 768 维向量
        S->>M: 存储向量 + 文本 + 元数据
    end

    S->>DB: 更新文档状态<br/>(status=READY)
    S-->>C: 返回文档信息
    C-->>U: 返回成功响应
```

## 💬 RAG 问答流程

```mermaid
sequenceDiagram
    participant U as 用户浏览器
    participant C as ChatController
    participant S as ChatService
    participant AI as KnowledgeAssistant<br/>(LangChain4j AiServices)
    participant E as EmbeddingModel<br/>(Ollama nomic-embed-text)
    participant M as Milvus
    participant LLM as ChatLanguageModel<br/>(Ollama Qwen2.5)
    participant DB as MySQL

    U->>C: POST /api/chat/send<br/>{conversationId, message}
    C->>C: 验证 JWT Token
    C->>S: chat(userId, conversationId, message)

    S->>DB: 保存用户消息

    S->>AI: 构建 RAG 助手
    AI->>E: 问题向量化
    E->>E: 调用 Ollama Embedding API
    E-->>AI: 返回 768 维向量

    AI->>M: 相似度检索<br/>(COSINE, Top 5)
    M-->>AI: 返回相关文本片段

    AI->>AI: 拼接 Prompt<br/>(系统指令 + 检索结果 + 用户问题)

    AI->>LLM: 调用 Qwen2.5 生成回答
    LLM->>LLM: POST http://localhost:11434/api/chat
    LLM-->>AI: 返回生成的回答

    AI-->>S: 返回回答
    S->>DB: 保存 AI 回复
    S-->>C: 返回回答
    C-->>U: 返回响应
```
